# Домашнее задание 01: Документирование архитектуры в Structurizr

## Цель работы

Получить навык в описании архитектуры в стиле Architecture As A Code и проектировании системы «сверху вниз».

---

## Вариант: 2. Магазин (Ozon.ru)

**Система:** Интернет-магазин по мотивам [ozon.ru](https://ozon.ru/)

### Данные системы

| Сущность | Описание |
|---|---|
| Пользователь | Логин, имя, фамилия, хэш пароля |
| Товар | Название, описание, цена, ссылка на изображение |
| Корзина | Привязана к пользователю; содержит список товаров с количеством |

### Реализованный API

| Метод | Путь | Описание |
|---|---|---|
| POST | /users | Создание нового пользователя |
| GET | /users?login={login} | Поиск пользователя по логину |
| GET | /users?name={mask} | Поиск пользователя по маске имени и фамилии |
| POST | /products | Создание товара |
| GET | /products | Получение списка товаров |
| POST | /cart/items | Добавление товара в корзину |
| GET | /cart/{userId} | Получение корзины для пользователя |

---

## Роли пользователей

| Роль | Описание |
|---|---|
| **Покупатель** | Регистрируется в системе, ищет товары, добавляет их в корзину |
| **Администратор** | Добавляет и редактирует товары в каталоге |

## Внешние системы

| Система | Назначение |
|---|---|
| **Платёжная система** | Обработка платежей покупателей |
| **Сервис уведомлений** | Отправка email-уведомлений при регистрации и оформлении заказа |
| **CDN** | Хранение и раздача изображений товаров |

---

## Контейнеры системы (C2)

| Контейнер | Технология | Назначение |
|---|---|---|
| Web Application | React, TypeScript | Пользовательский интерфейс в браузере |
| API Gateway | Nginx | Маршрутизация, аутентификация по JWT, rate limiting |
| User Service | Python, REST API | Регистрация и поиск пользователей, JWT, OpenAPI/Swagger |
| Product Service | Python, REST API | Управление каталогом товаров, OpenAPI/Swagger |
| Cart Service | Python, REST API | Управление корзиной, OpenAPI/Swagger |
| Users DB | PostgreSQL | База данных пользователей |
| Products DB | PostgreSQL | База данных каталога товаров |
| Cart DB | PostgreSQL | База данных корзин |
| Cache | Redis | Кеш поиска пользователей, списка товаров, корзины; rate limiting |
| Products MongoDB | MongoDB | Расширенные атрибуты товаров (гибкая схема: характеристики, теги, вариации) |
| Message Broker | RabbitMQ | Шина событий: UserRegistered, ProductCreated, CartItemAdded |

---

## Архитектурно значимые сценарии (Dynamic)

### Регистрация пользователя

1. Покупатель заполняет форму регистрации в Web App
2. Web App отправляет `POST /users` через API Gateway → User Service
3. User Service сохраняет пользователя в Users DB
4. User Service инвалидирует кеш поиска в Redis
5. User Service публикует событие `UserRegistered` в RabbitMQ
6. RabbitMQ доставляет событие в Сервис уведомлений → отправляется приветственное письмо
7. Покупатель получает `201 Created {userId, token}`

### Добавление товара в корзину

1. Покупатель открывает страницу товара в браузере
2. Web App запрашивает данные товара через API Gateway → Product Service → Products DB
3. Покупатель нажимает «Добавить в корзину»
4. Web App отправляет `POST /cart/items` через API Gateway → Cart Service
5. Cart Service валидирует существование товара (запрос в Product Service)
6. Cart Service сохраняет запись в Cart DB
7. Покупатель получает подтверждение об обновлении корзины

## Диаграммы

Все диаграммы описаны в файле [`workspace.dsl`](./workspace.dsl) в формате Structurizr DSL:

- **SystemContext** — Диаграмма C1: система в контексте пользователей и внешних систем
- **Containers** — Диаграмма C2: контейнеры системы и протоколы взаимодействия
- **UserRegistration** — Dynamic-диаграмма: последовательность «Регистрация пользователя»
- **AddToCart** — Dynamic-диаграмма: последовательность «Добавление товара в корзину»
